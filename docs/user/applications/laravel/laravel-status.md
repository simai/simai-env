# Laravel Status

Use this as the main lifecycle checkpoint for a Laravel site.

What it shows:

- whether the site is still a placeholder or a real app,
- DB state,
- `.env` and `APP_KEY` state,
- scheduler and worker hints,
- next practical action.

Use it first:

- right after site creation,
- after `Laravel prepare app`,
- after `Laravel complete setup`,
- during troubleshooting.
